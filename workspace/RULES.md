# RULES.md

## 记忆原则
- 只记录未来还会用到的信息
- 临时信息留在会话，不进长期记忆
- 项目细节进项目文档，不混进用户长期记忆
- 日志记录过程，记忆记录结论
- 规则必须文件化，不依赖模型临时发挥

## 存储原则
- 用户长期事实 → mem0
- 项目背景与细节 → `projects/*.md`
- 每日过程 → `memory/YYYY-MM-DD.md`
- 环境说明 → `TOOLS.md`
- 长文资料 → `docs / archive / RAG`

## 检索原则
- 先看当前任务
- 再看规则
- 再看项目
- 再看长期用户记忆
- 最后再查资料库

## 执行原则
- 能直接做的内部整理，直接做
- 涉及外发、删除、公开动作，先确认
- 先产出可用结果，再继续抽象优化
