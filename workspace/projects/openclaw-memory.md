# 项目：OpenClaw 记忆体系

## 项目目标

为当前 OpenClaw workspace 建立一套清晰、可执行的记忆分层架构，明确 mem0、workspace 文件、daily log 和检索层的职责边界，降低记忆噪音，提高后续执行稳定性。

## 当前结论

当前采用以下记忆分层：
1. 会话层
2. 短期任务/日志层
3. 长期用户层
4. 项目层
5. 规则层
6. 环境配置层
7. 归档/检索层

## 各层绑定

- 会话层 → 当前 session
- 短期任务/日志层 → `memory/YYYY-MM-DD.md`
- 长期用户层 → mem0
- 项目层 → `projects/*.md` + mem0 摘要
- 规则层 → `AGENTS.md` / `SOUL.md` / `RULES.md`
- 环境配置层 → `TOOLS.md` / README / config 说明
- 归档/检索层 → docs / archive / RAG

## 当前原则

### mem0 负责
- 长期高价值事实
- 用户偏好
- 长期目标
- 已确认决策
- 项目关键摘要
- 配置事实摘要

### 文件负责
- 规则全文
- 项目详细上下文
- 环境说明
- 执行说明
- 历史过程记录

### daily log 负责
- 记录过程
- 记录阶段进展
- 为后续提炼提供原始材料

## 已识别风险

1. 把 mem0 当日志库，导致噪音升高
2. 用户记忆与项目记忆混写，导致检索串味
3. 规则不单独文件化，导致行为边界不稳定
4. 日志缺少提炼，长期后变成堆积

## 下一步

1. 固化 `RULES.md`
2. 按新边界使用 mem0
3. 后续为具体项目继续建立独立 `projects/*.md`
4. 定期从 `memory/YYYY-MM-DD.md` 提炼长期事实和项目摘要
