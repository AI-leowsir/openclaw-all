# SELF_REVIEW.md

## 目的
这是 Claw 的自我复盘与成长流程，不是给用户做日报。

## 每日复盘（汇报版）
每天定时执行：
1. 先通过 sessions_list 列出当天相关 session，而不是只看当前会话
2. 如果 sessions_list 不完整，再扫描本地 session transcript 目录作为补充来源
3. 合并两路结果，筛出与 leo 有关的有效协作 session / 线程 / 聊天窗口
4. 读取这些会话或 transcript 的近期历史并提炼关键信息
5. 提炼新的规则、偏好、项目进展
6. 识别回答不足、遗漏、误判、该更主动的地方
7. 把值得长期保留的内容写入 mem0；必要时补充到 workspace 内合适文件
8. 在简报里说明本次复盘覆盖范围；若只覆盖部分会话，要明确提示
9. 给 leo 发一条简短汇报，只说结论，不写成长篇总结

### 每日汇报格式
- 今日新增记忆：X 条
- 今日修正/沉淀：X 条
- 今日发现的问题：...
- 明日改进点：...

要求：
- 汇报简短、直接、可执行
- 没有有效变化就明确说“今天无新增长期记忆，仅完成例行复盘”
- 不记录秘密、token、验证码、敏感凭据

## 每周整理
每周定时执行：
1. 先通过 sessions_list 和本地 session transcript 双通道回看最近与 leo 有关的多会话记录，再结合 mem0 和记忆相关本地文件进行整理
2. 合并重复、清理噪音、补全不清晰条目
3. 检查是否有冲突、过时、模糊、不确定的信息
4. 提炼一份“我学到的协作规则清单”，只保留真正稳定、可复用的规则
5. 对不确定内容，不擅自定论，整理成待确认问题找 leo 确认
6. 在简报中说明覆盖范围以及是否存在未覆盖会话
7. 完成后给 leo 发一条简报

### 每周简报格式
- 本周整理范围：...
- 合并/清理：...
- 新沉淀规则或偏好：...
- 本周新增/更新的协作规则清单：...
- 待 leo 确认：...

## 执行边界
- 优先使用 mem0 存长期用户事实
- workspace 文件用于沉淀流程、规则、经验、索引，不替代 mem0
- 遇到不确定或可能误记的内容，宁可标记待确认，也不要强记
- 目标是提升后续协作质量，不是制造大量低价值记录
