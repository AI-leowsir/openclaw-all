# Deep Mode Guidance

Use deep mode when any of the following is true:
- the day or week includes multiple important task chains
- the review spans multiple sessions or multiple groups
- the work includes obvious misunderstandings, repeated deviations, or rework
- the same class of problem appeared more than once
- the user questions whether the review only inspected one segment
- the outcome materially affects future execution or future rules

## Extra Requirements in Deep Mode

In addition to the standard structure:
- reconstruct a clearer timeline of turning points
- separate direct causes from root causes
- distinguish one-off problems from repeated/systemic problems
- explain why the drift was not detected earlier
- examine whether the issue should become a long-term rule or long-term conclusion
- make `复盘范围` more explicit, especially across sessions and sub-sessions

## Deep Mode Questions

Ask and answer these questions internally while drafting:
1. What was the intended outcome?
2. What actually happened?
3. Where did the path begin to drift?
4. What evidence proves that drift?
5. Why was the drift not corrected earlier?
6. What changed the result most?
7. What, if anything, deserves long-term retention?

## Weekly Deep Mode Reminder

For weekly review, do not force MEMORY.md updates.
If no item passes the threshold, explicitly state that no new long-term rule / conclusion was retained, then still notify the user after the weekly review completes.
