# 周复盘完成通知模板

用于周复盘执行完成后，向用户发送简短汇报。

## 目标
- 简洁
- 先给结论
- 只汇报对用户有用的信息
- 不重复整篇周复盘内容

## 固定结构
1. 周复盘是否完成
2. `MEMORY.md` 是否更新
3. 如有新增长期沉淀，只概述新增项
4. 如无新增长期沉淀，明确说明“本周无新增长期规则/长期结论”
5. 如有候选项，可简要提示目前仍在观察，不展开长解释

## 推荐写法
### 情况 A：已更新 `MEMORY.md`
本周周复盘已完成。
`MEMORY.md` 已更新。

新增长期沉淀：
- 长期规则：……
- 长期结论：……

仍在观察的候选项：
- ……

### 情况 B：未更新 `MEMORY.md`，但有候选项
本周周复盘已完成。
`MEMORY.md` 本周未更新。

本周无新增长期规则 / 长期结论。
仍在观察的候选项：
- ……

### 情况 C：未更新 `MEMORY.md`，也无候选项
本周周复盘已完成。
`MEMORY.md` 本周未更新。

本周无新增长期规则 / 长期结论。
本周也无新增候选项。

## 写作限制
- 不要重复周复盘正文
- 不要展开过程细节
- 不要写文件路径
- 不要写“去哪里查看详情”
- 保持 3~8 行为宜
- 允许项目符号，但不要写成长报告
